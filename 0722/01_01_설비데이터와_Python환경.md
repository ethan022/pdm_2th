## 1. 예측 정비(Predictive Maintenance)란

### 정비 방식 세 가지

| 방식 | 언제 정비하나 | 문제점 |
|---|---|---|
| 사후 정비 | **고장 난 뒤에** | 이미 라인이 멈춤. 손실이 큼 |
| 예방 정비 | **정해진 주기마다** | 멀쩡한 부품도 교체 → 낭비 |
| **예측 정비** | **고장 징후가 보일 때** | 데이터와 분석 능력이 필요 |

우리가 배우는 건 세 번째, **데이터로 고장 징후를 미리 읽는 것**입니다.

### 데이터에서 정비 판단까지 5단계

```
센서 측정  →  데이터 수집  →  전처리·분석  →  이상 탐지  →  정비 판단
```

지금 배우는 파이썬 기초는 이 중 **3단계(전처리·분석)** 를 하기 위한 도구입니다.

## 2. 설비 데이터의 생김새

- 설비 데이터는 대부분 **숫자**입니다 — 온도, 진동, 압력, 회전수, 전류…
- 센서가 **시간 순서대로** 값을 쌓습니다. (1초마다, 1분마다)
- 하지만 숫자만 있는 게 아닙니다. **글자 데이터**도 잔뜩 섞여 있습니다.
  - 센서 이름 — `temp_sensor_1`, `vibration_02`
  - 상태 라벨 — `NORMAL`, `정상`, `WARNING`
  - 설비 코드 — `EQP-001`, `PUMP_A_03`
  - 파일·컬럼명 — `cmapss_fd001.csv`, `cycle`

> 그래서 3단원에서 **문자열 다루기**를 배웠습니다. 숫자 분석 전에 반드시 거치는 단계입니다.

### 이상 징후의 두 가지 모양

1. **값이 갑자기 튄다** — 순간적으로 확 올라감
2. **값이 서서히 변한다** — 조금씩 계속 올라감 (이쪽이 더 놓치기 쉬움)

## 5. Git과 GitHub

### 왜 필요한가 — "파일명 지옥"

```
최종.py
최종_수정.py
최종_수정_진짜최종.py
최종_수정_진짜최종_v2.py   ← 뭐가 최신이지?
```

이 방식의 세 가지 한계:

1. **어느 게 최신인지 모른다**
2. **무엇이 바뀌었는지 모른다**
3. **여럿이 같이 작업할 수 없다**

Git은 파일을 복사하는 대신 **스냅샷(변경 시점)** 을 기록합니다.
→ 언제든 과거로 되돌아갈 수 있고, 무엇이 바뀌었는지 볼 수 있습니다.

### 파일의 4가지 상태

```
수정 안 함(Unmodified) → 수정함(Modified) → 스테이징(Staged) → 커밋됨(Committed)
                            git add ↗          git commit ↗
```

### 기본 명령어

```bash
# --- 처음 한 번만 ---
git config --global user.name "내이름"
git config --global user.email "내이메일@example.com"

git init                                    # 이 폴더를 저장소로 만들기 (.git 폴더 생성)

# --- 매번 반복하는 사이클 ---
git status                                  # 지금 상태 확인 (가장 자주 씀)
git add 파일명                               # 스테이징에 올리기
git add .                                   # 전부 올리기
git commit -m "메시지"                       # 스냅샷 저장
git log                                     # 지금까지의 커밋 기록
git log --oneline                           # 한 줄로 간단히

# --- GitHub(원격 저장소)와 주고받기 ---
git remote add origin https://github.com/아이디/저장소이름.git
git push -u origin main                     # 처음 올릴 때
git push                                    # 그 다음부터
git pull                                    # 원격의 변경사항 가져오기
```

### 좋은 커밋 메시지

- ❌ `수정`, `ㅇㅇ`, `asdf`
- ✅ `변수 실습 코드 추가`, `문자열 슬라이싱 예제 정리`

**무엇을 했는지**가 드러나게 씁니다.

### push와 pull 방향

```
내 컴퓨터(로컬)  ──push──▶  GitHub(원격)
내 컴퓨터(로컬)  ◀──pull──  GitHub(원격)
```

### .gitignore

올리면 안 되는 / 올릴 필요 없는 파일을 적어두는 파일입니다.

```
__pycache__/
.ipynb_checkpoints/
*.log
.env            ← 비밀번호·API 키가 든 파일은 절대 올리지 않기
```
