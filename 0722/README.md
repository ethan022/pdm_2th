## 3. 한 장 치트시트

### 출력

```python
print("글자")            # 글자는 따옴표로
print(75)                # 숫자는 따옴표 없이
print("온도:", 75)        # 쉼표로 여러 개 → 사이에 자동 띄어쓰기
print(75 + 5)            # 계산 후 출력 → 80
# 샵 뒤는 주석 (컴퓨터가 실행하지 않음)
```

### 변수

```python
temp = 25          # = 는 '같다'가 아니라 '저장하라'
temp = 30          # 재할당 → 마지막 값만 남음
x = x + 5          # 오른쪽을 먼저 계산해서 다시 x에 저장
a, b = 10, 20      # 한 줄에 여러 개 (개수가 같아야 함)
b = a              # '그 순간의 값'을 복사 (이후 a가 바뀌어도 b는 그대로)
```

이름 규칙: 영문·숫자·밑줄만 / 숫자로 시작 불가 / 공백 불가 / 대소문자 구분 (`temp` ≠ `Temp`)
좋은 이름: `a` 보다 `sensor_count` — 소문자와 밑줄로 통일(snake_case)

### 자료형 4가지

| 자료형 | 예시 | 구분하는 법 |
|---|---|---|
| `int` 정수 | `3` | 따옴표·소수점 없음 |
| `float` 실수 | `25.5` | 소수점 있음 |
| `str` 문자열 | `"센서A"` | 따옴표로 감쌈 |
| `bool` 불린 | `True` | True / False (첫 글자 대문자!) |

판단 순서: **따옴표 → True/False → 소수점**
종류가 의심되면 추측하지 말고 `print(값)` → `type(값)` 두 단계 점검.

```python
print(3 + 5)        # 8      숫자끼리는 계산
print("3" + "5")    # 35     글자끼리는 이어 붙이기
print(3 + "5")      # TypeError
print(0.1 + 0.2)    # 0.30000000000000004   ← 실수는 작은 오차가 생길 수 있음
```

### 연산자

| 종류 | 기호 | 비고 |
|---|---|---|
| 산술 | `+` `-` `*` `/` | `/` 결과는 **항상 실수** (`10 / 2` → `5.0`) |
| 몫·나머지 | `//` `%` | `7 // 2` → `3`, `7 % 2` → `1` |
| 거듭제곱 | `**` | `2 ** 3` → `8` |
| 복합 할당 | `+=` `-=` | `score += 10` 은 `score = score + 10` |
| 비교 | `==` `!=` `>` `<` `>=` `<=` | 결과는 항상 `True` / `False` |
| 논리 | `and` `or` `not` | and 둘 다 · or 하나라도 · not 뒤집기 |

우선순위: `**` > `*` `/` `//` `%` > `+` `-` — 헷갈리면 괄호를 쓰면 된다.
문자열에도 쓸 수 있다: `"안녕" + "하세요"` → 안녕하세요, `"-" * 10` → `----------`

### 입력 → 변환 → 계산 → 출력

```python
age = int(input("나이: "))   # input 결과는 항상 문자열! → int()로 변환
print(age + 1)
```

- `int()` 글자 → 정수 &nbsp;·&nbsp; `float()` 글자 → 실수 &nbsp;·&nbsp; `str()` 숫자 → 글자
- `int("25.5")` 는 **오류**. 소수점이 있으면 `float()` 를 씁니다.
- `int(input("..."))` 는 괄호가 두 겹 — 여는 괄호 두 개면 닫는 괄호도 두 개.

## 4. 자주 만나는 오류

| 오류 | 메시지 예시 | 원인 | 해결 |
|---|---|---|---|
| `NameError` | `name 'temp' is not defined` | 만든 적 없는 이름 / 따옴표 빠뜨림 / 대소문자 다름 | 변수를 먼저 만들거나 따옴표를 붙임 |
| `SyntaxError` | `invalid syntax` | 따옴표·괄호 **짝**이 안 맞음 | 여는 것과 닫는 것 개수 확인 |
| `TypeError` | `can only concatenate str...` | 글자 + 숫자를 그냥 더함 | 숫자를 `str()` 로 감싸기 |
| `ValueError` | `invalid literal for int()` | `int("스물다섯")` | 숫자 형태의 글자만 변환 |

### 오류 대처 4단계

1. **당황하지 않기** — 오류는 실패가 아니라 과정입니다.
2. **종류와 위치 읽기** — 맨 아랫줄의 `~Error` 와 `line 숫자`.
3. **그 줄과 위아래 줄 확인** — 짝이 맞는지, 이름이 맞는지.
4. **고치고 다시 실행.**

그래도 모르겠으면 → 빨간 메시지 **맨 아랫줄을 그대로 복사해서 검색**하세요.
