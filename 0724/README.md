## 2. 오늘의 핵심 두 줄

> **① 문자열 메서드는 원본을 바꾸지 않는다. 결과를 반드시 다시 받아야 한다.**

```python
s = "python"
s.upper()          # 결과가 그냥 버려짐
print(s)           # python  ← 안 바뀜!
s = s.upper()      # ← 이 패턴을 통째로 외우기
```

> **② 텍스트 정리 5단계 — 필요한 단계만 골라 쓰면 된다.**

```
공백제거   →   통일·치환      →   나누기   →   조각정리     →   합치기·출력
strip          lower·replace     split        조각 strip       join·f-string
```

## 3. 한 장 치트시트

### 03_04 대소문자

| 메서드 | 하는 일 | 예 |
|---|---|---|
| `upper()` / `lower()` | 모두 대문자 / 소문자 | `"NORMAL".lower()` → `normal` |
| `capitalize()` | 문장 첫 글자만 대문자 (**나머지는 소문자로!**) | `"PYTHON IS".capitalize()` → `Python is` |
| `title()` | 단어마다 첫 글자 대문자 | `"kim chul soo".title()` → `Kim Chul Soo` |
| `isupper()` / `islower()` | 모두 대문자/소문자인지 **검사** (True/False) | `"Abc".isupper()` → `False` |

대소문자 무시 비교: `a.lower() == b.lower()` — 표준 패턴.

### 03_04 공백

| 메서드 | 하는 일 | 예 (`s = "  대기  "`) |
|---|---|---|
| `strip()` | 앞뒤 공백 제거 | `[대기]` |
| `lstrip()` / `rstrip()` | 왼쪽만 / 오른쪽만 | `[대기  ]` / `[  대기]` |
| `strip("#$")` | 앞뒤에서 그 글자들 제거 | `"#$#대기#$#"` → `대기` |

- 공백은 `"[" + s + "]"` 로 감싸서 눈으로 확인하는 습관.
- **`strip` 은 앞뒤 전용** — 가운데는 못 지움 → `replace` 로.

### 03_05 치환·분리·결합

| 메서드 | 용도 | 결과 |
|---|---|---|
| `replace(A, B)` | A를 B로 바꾸기 (**전부 한 번에**) | 문자열 |
| `replace(A, "")` | A를 제거 (중간 공백도!) | 문자열 |
| `split()` | 공백 기준으로 나누기 (여러 칸 OK) | 리스트 |
| `split(",")` | 구분자 기준으로 나누기 | 리스트 |
| `split(",", 1)` | 앞에서 1번만 나누기 | 리스트 |
| `"구분자".join(리스트)` | 조각 합치기 (**구분자가 앞!**) | 문자열 |

```python
"정 상 가 동".replace(" ", "")          # '정상가동'   strip이 못 하는 중간 공백
int("3,000".replace(",", ""))          # 3000        쉼표 제거 후 숫자 변환
"2025-01-15".split("-")                # ['2025', '01', '15']  → parts[0] 은 '2025'
"-".join(["2025", "01", "15"])         # '2025-01-15'
"-".join(raw.split("/"))               # 구분자 통째로 교체 (split ↔ join 짝꿍)
raw.strip().split(",")                 # 파일 한 줄은 strip 먼저, split 나중
parts[1].strip().lower()               # 나눈 조각도 따로 정리
```

- **replace vs split** — 바꿔서 하나로 둘 거면 `replace`, 나눠서 골라 쓸 거면 `split`.
- `join` 대상은 **문자열 조각만**. 숫자가 섞이면 `TypeError` → `str()` 로 먼저 변환.
- `print(2025, 1, 15, sep="-")` → `2025-1-15` (출력할 때 구분자), `end` 는 끝 문자.

### 03_06 f-string

```python
name = "PUMP_A"
temp = 87.456

f"설비 {name}"            # 변수 끼워넣기 (따옴표 앞 f 필수!)
f"온도 {temp}도"           # 숫자도 str() 없이 자동 변환
f"{2 * 60}분"             # 중괄호 안에서 계산 가능
f"{temp:.1f}"             # 87.5  ← 소수점 1자리, 반올림
f"{temp:.2f}"             # 87.46
f"[{name}] 값: {temp:.1f}"  # 중괄호 바깥 기호는 양식 그대로
```

**f를 빠뜨리면** → `print("이름 {name}")` 은 `이름 {name}` 그대로 출력됩니다.
중괄호와 변수 이름이 그대로 보이면 따옴표 앞 `f` 부터 확인하세요.

### 실전 정리 사례 (03_06)

| 데이터 | 패턴 |
|---|---|
| 이메일 | `email.strip().lower()` |
| 전화번호 | `phone.replace("-", "").replace(" ", "")` → 슬라이싱+f-string으로 재형식 |
| 이름 | `name.strip().title()`, 공백 들쭉날쭉이면 `" ".join(name.split())` |
| 컬럼명 | `col.strip().lower().replace(" ", "_")` |
| 라벨 통일 | `label.strip().lower()` |
| CSV 한 줄 | `raw.strip().split(",")` → 조각별 `strip()` → `float()` → f-string 리포트 |

## 4. 자주 만나는 오류

| 오류 | 원인 | 해결 |
|---|---|---|
| 값이 안 바뀜 (오류 아님!) | `s.strip()` 결과를 안 받음 | `s = s.strip()` |
| `{name}` 이 그대로 출력 | 따옴표 앞 `f` 빠뜨림 | `f"...{name}..."` |
| `ValueError` | `int("3,000")` — 쉼표는 숫자가 아님 | `replace(",", "")` 후 변환 |
| `TypeError` | `"-".join(["a", 1])` — 숫자 섞임 | `str(1)` 로 먼저 변환 |
| 조각에 공백 남음 | `"a, b".split(",")` → `' b'` | 꺼낸 조각에 `.strip()` |

## 5. 스스로 점검

- [ ] `s.upper()` 만 쓰면 왜 안 바뀌는지 설명할 수 있다
- [ ] `strip` 과 `replace(" ", "")` 의 차이(앞뒤 vs 전부)를 안다
- [ ] `"3,000"` 을 숫자로 바꾸는 코드를 쓸 수 있다
- [ ] `split` 결과가 리스트라는 것과, 번호로 조각을 꺼낼 수 있다는 것을 안다
- [ ] `"-".join(parts)` 에서 구분자가 앞에 온다는 것을 외웠다
- [ ] split ↔ join 조합으로 구분자를 통째로 바꿀 수 있다
- [ ] f-string 에서 `f` 를 빠뜨리면 어떻게 되는지 안다
- [ ] `{value:.2f}` 가 반올림된 소수점 2자리라는 것을 안다
- [ ] 텍스트 정리 5단계 흐름을 순서대로 말할 수 있다
