## 2. 오늘의 핵심 두 줄

> **① 리스트 메서드는 원본을 직접 바꾼다 — 문자열과 반대!**

```python
s = s.upper()        # 문자열: 결과를 다시 받아야 함 (03_04)
temps.sort()         # 리스트: 이걸로 끝. 원본이 이미 바뀜
temps = temps.sort() # ❌ 이러면 temps가 None이 된다!
```

> **② 조건문은 위에서부터 검사해서 '처음 참인 곳 하나만' 실행한다.**

```python
if score >= 90:      # 좁은(까다로운) 조건을 위에
    print("우수")
elif score >= 60:    # 넓은 조건을 아래에
    print("보통")
```

## 3. 한 장 치트시트 — 04_01 리스트

### 만들기와 기본

```python
temps = [25, 26, 24, 28, 27]     # 대괄호 + 쉼표
machines = ["펌프", "모터"]        # 글자는 따옴표 필수
empty = []                       # 빈 리스트 (모아갈 때 시작점)
len(temps)                       # 5 — 담긴 값의 개수
```

### 인덱싱·슬라이싱 — 문자열과 같은 규칙

```python
temps[0]      # 25          첫 번째 (0부터!)
temps[-1]     # 27          마지막
temps[5]      # IndexError  (5개면 인덱스는 0~4)
temps[1:4]    # [26, 24, 28]   시작 포함, 끝 제외
temps[:3]     # 앞 3개       temps[-3:]  # 뒤 3개
temps[::2]    # 2칸씩
temps[2] = 99 # 그 자리만 교체 (문자열은 안 되지만 리스트는 됨!)
```

### 메서드 정리

| 용도 | 메서드 | 하는 일 |
|---|---|---|
| 추가 | `append(값)` | 맨 끝에 하나 |
| | `insert(위치, 값)` | 그 위치에 끼워 넣기 |
| | `extend(리스트)` | 다른 리스트를 풀어서 이어붙이기 |
| 제거 | `remove(값)` | 그 **값**의 첫 번째 항목 제거 |
| | `pop(위치)` | 그 **위치**의 값을 **꺼내 돌려주며** 제거 |
| | `del temps[위치]` | 위치로 삭제 (점 없이, 반환 없음) |
| 정렬 | `sort()` / `sort(reverse=True)` | 오름차순 / 내림차순 |
| | `reverse()` | 크기 무관, 현재 순서 뒤집기 |
| 탐색 | `count(값)` / `index(값)` | 개수 / 처음 위치 |
| 확인 | `값 in temps` | 들어 있는지 True/False |

- 기준이 **값**이면 `remove`, **위치**면 `pop`/`del`. 지운 값을 써야 하면 `pop`.
- `remove`는 **첫 번째 것만** 지운다.
- 정렬 후 `temps[0]` = 최솟값, `temps[-1]` = 최댓값.

## 4. 한 장 치트시트 — 04_02 조건문

### 기본 구조

```python
if 조건식:            # 콜론 = 블록 시작 신호
    실행할 코드        # 들여쓰기(4칸) = 조건에 속한 블록
elif 다른조건:        # 앞 조건이 거짓일 때만 검사
    실행할 코드
else:                # 나머지 전부 (조건식 없음)
    실행할 코드
그다음코드             # 들여쓰기 밖 = 항상 실행
```

### 경계 표현 — 판정이 갈리는 지점

| 우리말 | 기호 | 기준값 자체는? |
|---|---|---|
| 이상 | `>=` | **포함** |
| 초과 | `>` | 미포함 |
| 이하 | `<=` | **포함** |
| 미만 | `<` | 미포함 |

### 조건 엮기

```python
if temp >= 20 and temp <= 60:    # 둘 다
if 20 <= temp <= 60:             # ↑ 같은 뜻 (연쇄 비교 — 새로 배운 표기)
if temp > 60 or pressure > 7:    # 하나라도
if not is_running:               # ~가 아니면
```

## 5. 오늘의 함정 모음

| 함정 | 증상 | 예방 |
|---|---|---|
| `temps = temps.sort()` | 리스트가 `None`이 됨 | 리스트 메서드는 원본을 바꿈 — 다시 받지 않기 |
| elif 순서 (넓은 조건 먼저) | 95점이 "보통" — **오류도 안 남** | 높은 기준부터 위에 배치 |
| 들여쓰기 풀림 | 블록 밖 줄이 **항상** 실행됨 — 오류 없음 | 블록에 속한 줄인지 항상 의식 |
| `if x = 5:` | `SyntaxError` | 조건식에는 `==` |
| "이상"인데 `>` 사용 | 경계값(정확히 19살)에서 판정 틀림 | 이상/이하는 `>=` `<=` |
| `remove`가 다 지운 줄 앎 | 중복 중 첫 번째만 사라짐 | 개수는 `count`로 확인 |
| `temps[len(temps)]` | `IndexError` | 마지막 인덱스는 `개수-1`, 그냥 `[-1]` 쓰기 |
