## 2. 왜 문자열을 배우나

설비 데이터는 숫자뿐 아니라 **글자 데이터로 가득**합니다.

- 센서 이름 — `temp_sensor_1`, `vibration_02`
- 상태 라벨 — `NORMAL`, `정상`, `WARNING`
- 설비 코드 — `EQP-001`, `PUMP_A_03`
- 파일·컬럼명 — `cmapss_fd001.csv`, `cycle`

문자열 정리는 **숫자 분석 전에 반드시 거치는 단계**입니다.
오늘 배운 `in` / `find` 는 다음 단원 **데이터 필터링**의 기초 개념이기도 합니다.

## 4. 한 장 치트시트

### 문자열 만들기

```python
name = '홍길동'                  # 작은따옴표
status = "정상"                  # 큰따옴표 — 둘 다 가능, 시작과 끝만 맞추면 됨
print("It's normal")            # 안에 ' 이 있으면 바깥은 "
print('그는 "정상"이라 했다')       # 안에 " 이 있으면 바깥은 '

notice = """설비 점검 안내
1. 전원 확인
2. 센서 점검"""                  # 삼중 따옴표 = 여러 줄 그대로
```

### 이스케이프 문자

| 표기 | 의미 | 예 |
|---|---|---|
| `\n` | 줄바꿈 | `"1줄\n2줄"` |
| `\t` | 탭(간격) | `"이름\t상태"` |
| `\\` | 역슬래시 | `"C:\\data"` |

`\n` 은 화면엔 두 글자로 보여도 **한 글자**입니다. (`len("1줄\n2줄")` → 5)

### 빈 문자열 vs 공백 문자열

```python
len("")     # 0   글자가 0개
len(" ")    # 1   공백 글자 1개  ← 컴퓨터에겐 완전히 다른 값!
```

### 문자열과 숫자

```python
25 + 5          # 30    숫자: 합산
'25' + '5'      # 255   글자: 연결
'온도: ' + 25    # TypeError  →  '온도: ' + str(25)
```

### 인덱싱 — 글자 하나 꺼내기

```
글자    P    Y    T    H    O    N
양수    0    1    2    3    4    5     ← 첫 글자는 0번!
음수   -6   -5   -4   -3   -2   -1     ← 뒤에서부터
```

```python
word = "PYTHON"
word[0]      # 'P'   첫 글자
word[-1]     # 'N'   마지막 글자 (글자 수를 몰라도 됨)
word[6]      # IndexError  ← 6글자지만 번호는 0~5까지
```

### 슬라이싱 — 구간 잘라내기

```python
word[1:4]    # 'YTH'    시작 포함, 끝 제외 ★가장 중요★
word[:3]     # 'PYT'    처음부터
word[3:]     # 'HON'    끝까지
word[:]      # 'PYTHON' 전체
word[-3:]    # 'HON'    뒤 3글자
word[:-1]    # 'PYTHO'  마지막 뺀 나머지
word[::2]    # 'PTO'    두 칸씩
word[::-1]   # 'NOHTYP' 뒤집기
word[0:100]  # 'PYTHON' 범위를 넘어도 오류 없음 (가능한 만큼만)
```

**인덱싱 vs 슬라이싱** — 대괄호 안에 **콜론이 있으면 슬라이싱**, 없으면 인덱싱.

| | 인덱싱 | 슬라이싱 |
|---|---|---|
| 꺼내는 것 | 글자 한 개 | 구간 (여러 글자) |
| 범위 초과 | `IndexError` | 오류 없음 |

### 검색 도구

| 도구 | 하는 일 | 결과 | 못 찾으면 |
|---|---|---|---|
| `len(s)` | 글자 수 (공백 포함) | 숫자 | — |
| `s.count("a")` | 몇 번 나오는지 | 숫자 | `0` |
| `s.find("a")` | 처음 나오는 위치 | 숫자 | `-1` |
| `s.index("a")` | 처음 나오는 위치 | 숫자 | **오류!** |
| `"a" in s` | 들어 있는지 | 참·거짓 | `False` |
| `"a" not in s` | 안 들어 있는지 | 참·거짓 | — |
| `s.startswith("a")` | 그걸로 시작하는지 | 참·거짓 | `False` |
| `s.endswith(".csv")` | 그걸로 끝나는지 | 참·거짓 | `False` |
| `s == "정상"` | 완전히 같은지 | 참·거짓 | `False` |

- 결과가 **숫자**인 것 — `len` `count` `find` `index`
- 결과가 **참·거짓**인 것 — `in` `startswith` `endswith` `==`
- `find` 와 `index` 의 유일한 차이 → 못 찾았을 때 `-1` 이냐 **오류**냐

### find + 슬라이싱 조합 (자주 쓰는 패턴)

```python
email = "hong@company.com"
at = email.find("@")     # 4
email[:at]               # 'hong'
email[at+1:]             # 'company.com'
```

### == 비교의 함정

```python
"NORMAL" == "normal"     # False   대소문자는 다른 글자
"정상" == "정상 "         # False   뒤 공백 하나 차이도 다른 값
```

같은 상태가 `NORMAL` / `Normal` / `normal` 로 섞이면 **따로 집계**됩니다.
→ 비교 전에 대소문자·공백을 통일해야 합니다. (7/24 `03_04 공백과 대소문자 정리`)

## 5. 자주 만나는 오류

| 오류 | 원인 | 해결 |
|---|---|---|
| `SyntaxError` | 따옴표 짝 불일치 — `'It's normal'` | 바깥을 다른 따옴표로 (`"It's normal"`) |
| `TypeError` | 글자 + 숫자 — `'온도: ' + 25` | `str(25)` 로 감싸기 |
| `ValueError` | 숫자 아닌 글자를 `int()` | 숫자 형태의 글자만 변환 |
| `IndexError` | 없는 번호를 인덱싱 — `"PYTHON"[6]` | 0부터 다시 세어보기 |
| `ValueError` | `"정상".index("고장")` | 없을 수도 있으면 `find()` 를 쓰기 |
